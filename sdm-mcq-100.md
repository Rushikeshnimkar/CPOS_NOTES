## Section 1: Git Version Control (Q1-15)

**Q1. [Easy]** Git is a:
- a) Centralized Version Control System
- b) Distributed Version Control System
- c) Local Version Control System
- d) Database Management System

**Answer: b) Distributed Version Control System**

---

**Q2. [Easy]** Which command creates a new Git repository?
- a) git clone
- b) git init
- c) git create
- d) git new

**Answer: b) git init**

---

**Q3. [Easy]** Which command stages all changes for commit?
- a) git commit -a
- b) git add .
- c) git stage all
- d) git push

**Answer: b) git add .**

---

**Q4. [Easy]** What does `git status` show?
- a) Commit history
- b) State of working directory and staging area
- c) Remote repositories
- d) Branch list

**Answer: b) State of working directory and staging area**

---

**Q5. [Easy]** Which command downloads a repository from remote?
- a) git init
- b) git pull
- c) git clone
- d) git fetch

**Answer: c) git clone**

---

**Q6. [Medium]** What does `git reset --hard HEAD~1` do?
- a) Creates a new commit
- b) Moves HEAD to previous commit and discards changes
- c) Stashes changes
- d) Creates a new branch

**Answer: b) Moves HEAD to previous commit and discards changes**

---

**Q7. [Medium]** Which command is used to apply a specific commit from another branch?
- a) git merge
- b) git rebase
- c) git cherry-pick
- d) git fetch

**Answer: c) git cherry-pick**

---

**Q8. [Medium]** What is the difference between `git fetch` and `git pull`?
- a) No difference
- b) fetch downloads changes, pull downloads and merges
- c) pull downloads changes, fetch downloads and merges
- d) fetch is for local, pull is for remote

**Answer: b) fetch downloads changes, pull downloads and merges**

---

**Q9. [Medium]** Which Git object type stores file contents?
- a) Tree
- b) Commit
- c) Blob
- d) Tag

**Answer: c) Blob**

---

**Q10. [Medium]** What does `git stash` do?
- a) Deletes changes
- b) Saves uncommitted changes temporarily
- c) Commits changes
- d) Pushes to remote

**Answer: b) Saves uncommitted changes temporarily**

---

**Q11. [Hard]** In Git Flow, which branch contains production-ready code?
- a) develop
- b) feature
- c) main/master
- d) hotfix

**Answer: c) main/master**

---

**Q12. [Hard]** A fast-forward merge happens when:
- a) There are conflicts
- b) Target branch has no new commits since source diverged
- c) Two branches are unrelated
- d) Force push is used

**Answer: b) Target branch has no new commits since source diverged**

---

**Q13. [Hard]** What does `git rebase -i HEAD~3` do?
- a) Merges last 3 commits
- b) Interactive rebase of last 3 commits
- c) Deletes last 3 commits
- d) Creates 3 branches

**Answer: b) Interactive rebase of last 3 commits**

---

**Q14. [Hard]** The command `git reflog` shows:
- a) Only pushed commits
- b) History of HEAD movements
- c) Remote branches
- d) File changes

**Answer: b) History of HEAD movements**

---

**Q15. [Hard]** Which commit message type indicates a bug fix?
- a) feat
- b) fix
- c) docs
- d) refactor

**Answer: b) fix**

## Section 2: Software Engineering Fundamentals (Q16-30)

**Q16. [Easy]** Software is:
- a) Only programs
- b) Programs + Data
- c) Programs + Data + Documentation
- d) Only documentation

**Answer: c) Programs + Data + Documentation**

---

**Q17. [Easy]** Which principle states "Don't Repeat Yourself"?
- a) KISS
- b) DRY
- c) SOLID
- d) YAGNI

**Answer: b) DRY**

---

**Q18. [Easy]** KISS stands for:
- a) Keep It Simple, Stupid
- b) Keep It Short, Simple
- c) Keep It Secure, Safe
- d) Keep It Structured, Simple

**Answer: a) Keep It Simple, Stupid**

---

**Q19. [Easy]** Software Crisis refers to:
- a) Hardware failures
- b) Problems in software development like delays and overruns
- c) Network issues
- d) Security breaches

**Answer: b) Problems in software development like delays and overruns**

---

**Q20. [Medium]** In SOLID principles, 'S' stands for:
- a) Simple Responsibility
- b) Single Responsibility
- c) Software Responsibility
- d) System Responsibility

**Answer: b) Single Responsibility**

---

**Q21. [Medium]** The 'O' in SOLID means:
- a) Open for modification
- b) Open for extension, closed for modification
- c) Only one interface
- d) Object-oriented

**Answer: b) Open for extension, closed for modification**

---

**Q22. [Medium]** Cyclomatic Complexity V(G) is calculated as:
- a) E + N - 2
- b) E - N + 2
- c) N - E + 2
- d) E × N

**Answer: b) E - N + 2** (E=edges, N=nodes)

---

**Q23. [Medium]** LOC stands for:
- a) Level of Complexity
- b) Lines of Code
- c) Logic of Code
- d) Length of Calculation

**Answer: b) Lines of Code**

---

**Q24. [Medium]** Software does NOT have which characteristic compared to hardware?
- a) It doesn't wear out
- b) It's intangible
- c) It suffers from physical degradation
- d) It's complex

**Answer: c) It suffers from physical degradation**

---

**Q25. [Hard]** Brooks' Law states:
- a) "The best code is no code"
- b) "Adding manpower to a late project makes it later"
- c) "All bugs are shallow given enough eyes"
- d) "Always optimize for performance"

**Answer: b) "Adding manpower to a late project makes it later"**

---

**Q26. [Hard]** CMMI Level 3 is called:
- a) Initial
- b) Managed
- c) Defined
- d) Optimizing

**Answer: c) Defined**

---

**Q27. [Hard]** YAGNI stands for:
- a) You Always Get New Ideas
- b) You Aren't Gonna Need It
- c) Your Application Gets New Input
- d) You Are Going Nowhere Indefinitely

**Answer: b) You Aren't Gonna Need It**

---

**Q28. [Hard]** Quality Assurance (QA) focuses on:
- a) Finding defects
- b) Preventing defects through process
- c) Testing products
- d) Customer support

**Answer: b) Preventing defects through process**

---

**Q29. [Hard]** McCall's quality model divides factors into:
- a) Internal and External
- b) Operation, Revision, Transition
- c) Functional and Non-functional
- d) Static and Dynamic

**Answer: b) Operation, Revision, Transition**

---

**Q30. [Hard]** Defect Density is calculated as:
- a) Defects × KLOC
- b) Defects / KLOC
- c) KLOC / Defects
- d) Defects + KLOC

**Answer: b) Defects / KLOC**
